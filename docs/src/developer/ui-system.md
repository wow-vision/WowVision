# UI System

## Overview

React-inspired declarative UI framework. Virtual element trees are generated by functions, then reconciled against real elements. Accessibility-first: TTS announcements, focus-driven navigation, live field updates.

## Files

- `core/ui/UIManager.lua` — Element type registry and factory
- `core/ui/Generator.lua` — Virtual DOM and reconciliation engine
- `core/ui/Window.lua` — Window types
- `core/ui/WindowManager.lua` — Window instance management
- `core/UIHost.lua` — Main UI orchestrator
- `core/ui/elements/` — Element implementations
- `core/ui/navigators/` — Navigation system
- `core/ui/input/` — Input handling

## Element Hierarchy

### UIElement (base)
- InfoClass integration for property management
- Event system: `events` table with handlers
- Focus/unfocus with TTS announcements
- Live fields: values announced when they change
- `getFocusString()` returns label + type string for TTS

### Widget (extends UIElement)
- Has a `value` with data binding support
- `bind` config connects to data sources (Property, Field, Method, Function)
- Events: `click`, `drag`, `valueChange`
- Subclasses: Button, Checkbox, Text, EditBox, ProxyWidget

### Container (extends UIElement)
- `direction`: "vertical", "horizontal", "tab", "control-tab", "grid"
- `wrap`: navigation wraps around edges
- Child management: `add(child, index)` / `remove(child)` / `getChildren()`

### Context Types
- **Context** — Base layout container
- **StackContext** — Stack semantics with push/pop
- **WindowContext** — Window lifecycle and escape handling
- **HorizontalContext** — control-tab direction
- **Panel** — Tab direction container

## Virtual DOM & Reconciliation

### Virtual Element Definition

```lua
gen:Element("MyElement", {
    regenerateOn = {
        events = { "PLAYER_LEVEL_UP" },
        values = function(props) return { level = UnitLevel("player") } end,
    }
}, function(props)
    return { "Text", label = "Level: " .. props.level }
end)
```

Virtual elements define:
- **Regeneration conditions:** WoW events, frame fields, dynamic values
- **Generator function:** Returns element specs (type + props + children)

### Reconciliation

`reconcile(realParent, oldTree, newTree)` diffs virtual trees against real elements:
- **Keyed matching:** Uses `key` prop for optimal reordering
- **Non-keyed:** Index-based matching with type checking
- `updateProps()` applies prop changes to existing real elements
- `build()` / `unbuild()` create/remove real elements as needed

### GeneratorPanel

Bridges the Generator with real element containers:
- `startingElement` — Virtual element config to render
- Tracks dirty virtual elements for selective regeneration
- `iterate()` called each frame to reconcile changes

## Window System

### Window Types

1. **FrameWindow** — Detects state from frame visibility
2. **CustomWindow** — Custom function for state detection
3. **ManualWindow** — Opened/closed programmatically
4. **EventWindow** — Opens/closes on WoW events
5. **PlayerInteractionWindow** — PLAYER_INTERACTION_MANAGER events

### UIHost (Orchestrator)

- Manages HorizontalContext for window stacking
- Creates navigator for focus/input
- Coordinates WindowManager and MenuManager
- Combat handling: deactivates input during combat

## Navigation System

### Navigator Node Types

1. **NavigatorElementNode** — Leaf nodes (widgets). Tracks field value changes.
2. **NavigatorContainerNode** — Parent with selection. Handles prev/next/up/down.
3. **NavigatorPreservingContainerNode** — Caches child nodes (for StackContext).
4. **NavigatorSyncedContainerNode** — Index-synced (for scroll frames).

### Live Fields

Elements can declare live fields that auto-announce via TTS when values change:

```lua
liveFields = {
    value = "focus",    -- Announce only when element is focused
    enabled = "always", -- Always announce, even when not focused
}
```

The navigator detects changes via `reconcileFields()` and queues TTS announcements.

## Input System

- `BindingSet` — Collection of active key bindings
- `ActivationSet` — Input sources per element
- `Binding` — Individual key binding with action type
- Focus activates bindings, unfocus deactivates

## Data Binding

Widgets connect to data sources via the `bind` prop:

```lua
{ type = "Property", target = obj, property = "health" }
{ type = "Field", target = obj, field = fieldInstance }
{ type = "Method", target = obj, readMethod = "get", writeMethod = "set" }
{ type = "Function", get = func, set = func }
```

`WowVision.dataBinding:create(config)` creates a DataBinding instance.

## Complete Data Flow

1. Window registered → WindowManager polls `isOpen()`
2. State change → `window:open()` creates WindowContext
3. If generated: builds GeneratorPanel with starting virtual element
4. WindowContext added to UIHost's HorizontalContext
5. Navigator created, bindings activated
6. Each frame: `GeneratorPanel:iterate()` → reconcile virtual vs real tree
7. User input → binding → navigator → element event handler
8. Value changes → live field detection → TTS announcement
